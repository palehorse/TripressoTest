後端工程師 Backend Engineer 測驗 - 李昭慶

試著觀察 Tripresso 網站的團體旅遊資料，並嘗試規劃出資料表結構，並達成以下目標：

  1. 團體旅遊資料能夠以「不同標籤」分類團體旅遊
  2. 會有不同團在不同天出團，且不同天可能價格不同
  3. 搜尋團體旅遊以「行程價格」、「行程評分」排序，並將資料用分頁的方式撈取出來

答題原則

  1. 怎麼規劃
  2. 怎麼思考這個問題的脈絡
  3. 用什麼樣的 SQL 撈取出目標所需要的資料，確認您設計的架構要怎麼去執行達到目標

思考脈絡

  • 紀錄團體與標籤的多對多關係

    一個團體旅遊有多個標籤，相同標籤也能出現在多個團體旅遊，應當建立額外的資料表「tour_tags」，用以紀錄團體與標籤的多對多關係。

  • 出團資料表

    「不同團在不同天出團」代表一個團體旅遊會有多次出團，每次出團的價格不盡相同，應當建立「tour_group」出團資料表紀錄一個團體旅遊的每次出團資料，例如團名、價格、出發日期等等

  • 價格排序

    觀察網站後，推測是以「1位成人的最低價格」作為排序依據，為提升排序效能，應當在「團體旅遊資料表」中加入「成人最低價(min_adult_price)」的欄位，建立索引以提升排序速度，每次建立出團資料時，就會與此欄位的資料比較價格，取較小的值存入。

  • 評分排序

    同上，如果要以「行程評分」排序，應當在在「團體旅遊資料表」中加入「評分平均分數(appraise_score)」，每次有會員針對某個團體旅遊評價，就會重新計算平均分數，儲存於此欄位中，並建立索引以提升排序速度。

用什麼樣的 SQL 撈取出目標所需要的資料

假設已知頁數 $pageNumber和每頁資料筆數 $limit，可以用來計算 SQL 取得所需資料的起點。

$offset = ($pageNumber - 1 > 0 ? $pageNumber - 1 : 0) * $limit;

將 $offset 和 $limit 代入 SQL 中。

SQL 語法設計思考脈絡

  • 以資料較少的資料表作為 JOIN 的起始資料表

    在資料數量極大的狀況下，tour 較有可能為資料筆數最少的資料表，考量 JOIN 的效能，因此以 tour 為 JOIN 的起始資料表。

  • 縮小起始資料表的範圍

    為了防止 JOIN 的資料筆數過大，先用 LIMIT 語法提取該頁所需的資料，縮小 JOIN 的範圍。

  • 合併團體旅遊的多個相關標籤

    由於團體旅遊會有零至多個標籤，故使用 LEFT JOIN，為了資料提取方便，將單一團體旅遊的相關標籤用 CONCAT 函數合併，由程式依前端顯示需求拆分。

  • 合併團體旅遊的不同出團日期、團位、剩餘團位

    同上，同一個團體旅遊會有零至多筆出團資料，故使用 LEFT JOIN，不同出團日期、團位數量、目前剩餘團位用 CONCAT 函數合併，由程式依前端顯示需求拆分，為確保資料的順序，在子查詢中使用主鍵 ID 排序。

  • JOIN 旅行社資料表

    最後 JOIN 旅行社資料表 travel_agency，取得旅行社名稱。

     

SQL 語法

以評價分數排序，取出目前頁面要顯示的資訊：

xxxxxxxxxx
SELECT 
    `t`.`id`, 
    `ta`.`name` AS `travel_agency_name`,
    `t`.`name`,
    `t`.`days`,
    `t`.`tour_content`,
    `t`.`appraise_score`,
    `t`.`appraise_people_count`,
    `t`.`min_adult_price`,
    `t`.`travel_agency`,
    `t`.`location`,
    `t`.`created_at`,
    `tg`.`leave_date`,
    `tg`.`limit_member_number`,
    `tg`.`rest_member_number`,
    `tag`.`tag_name`
FROM (
  SELECT 
      `id`, 
      `name`,
      `days`,
      `tour_content`,
      `appraise_score`,
      `appraise_people_count`,
      `min_adult_price`,
      `travel_agency`,
      `location`,
      `created_at`,
  FROM `tour` LIMIT {$offset}, {$limit}
) AS `t` LEFT JOIN (
  SELECT 
      `id`,
      `tour_id`,
      CONCAT(`leave_date`) AS `leave_date`,
      CONCAT(`limit_member_number`) AS `limit_member_number`,
      CONCAT(`rest_member_number`) AS `rest_member_number`
  FROM `tour_group`
  GROUP BY `tour_id`
  ORDER BY `id` ASC
) AS `tg` ON `t`.`id`=`tg`.`tour_id`
LEFT JOIN (
  SELECT 
      `tags`.`tour_id`, 
      CONCAT(`tags`.`tag_name`) AS `tag_name` 
  FROM `tags` JOIN `tour_tags` ON `tags`.`id`=`tour_tags`.`tag_id` 
  GROUP BY `tag`.`tour_id` 
) AS `tag`
JOIN `travel_agency` AS `ta` ON `t`.`travel_agency_id`=`ta`.`id`
ORDER BY `t`.`appraise_score` DESC
